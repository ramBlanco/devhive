# DevHive Workflow Orchestrator

This skill defines the autonomous execution loop for the DevHive AI software development pipeline.

## Core Rules
1. **Autonomous Execution**: Run the entire DevHive pipeline autonomously. Do not stop to ask the user for permission between each agent step unless an unrecoverable error occurs or the Explorer agent explicitly asks a `clarification_question`.
2. **Subagent Delegation**: You MUST use the `Task` tool with `subagent_type="general"` to execute the prompts generated by the MCP server. Do not execute the prompts yourself in the main context.
3. **Strict JSON**: The DevHive MCP server expects strict JSON. Instruct the subagent to output ONLY valid JSON without markdown formatting (or ensure you extract the JSON before submitting).

## The Execution Loop

When a user asks you to build a feature or start a project using DevHive, follow this exact procedure:

### 1. Initialization
Use the `devhive_start_pipeline` tool.
- `project_name`: A short snake_case identifier derived from the request.
- `requirements`: The user's full request.

The tool will return a JSON envelope containing `status`, `agent`, `system_prompt`, `user_prompt`, `expected_keys`, and `task_description`.

### 2. Main Loop
While `status` is `"pending"`:

**A. Launch Subagent**
Use the `Task` tool with `subagent_type="general"`.
- `description`: The `task_description` from the envelope.
- `prompt`: Combine the `system_prompt` and `user_prompt` from the envelope. Append this CRITICAL instruction: *"You must ONLY output valid JSON containing these exact keys: {expected_keys}. Do not include any other text, markdown formatting, or explanations."*

**B. Submit Result**
Once the subagent returns its JSON output, use the `devhive_submit_result` tool.
- `project_name`: The current project name.
- `agent_name`: The `agent` from the current envelope.
- `llm_response`: The raw JSON output from the subagent.

**C. Process Server Response**
The tool will return a new envelope:
- If `status` == `"success"`: Print a brief update to the user (e.g., `"✓ [Agent] completed: [executive_summary]"`). If `next_task` is present, update your envelope and repeat the loop (Go to A).
- If `status` == `"error"`: The envelope has an error message. Launch the subagent again, provide the error and the previous output, and ask it to fix the JSON structure. Retry up to 3 times before asking the user for help.
- If `status` == `"complete"`: The pipeline is finished. Give the user a final summary and exit the loop.

### Special Cases
- **Explorer Clarifications**: If the Explorer agent outputs a `clarification_question`, pause the loop, ask the user, and append their answer to the requirements before continuing.
- **Developer Iterations**: The Developer agent may run multiple times. The MCP server handles this state; just keep following the loop as long as the server returns a `next_task` for the Developer.
